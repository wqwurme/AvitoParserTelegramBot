# 📖 Документация — Avito Telegram Bot

Бот следит за новыми объявлениями на Avito и присылает уведомления в Telegram в формате:

```
🏷 Название товара
💰 Цена ₽
🔗 Ссылка на объявление
```

---

## 📁 Структура проекта

```
avito_bot/
├── main.py           # Точка входа, хэндлеры бота, планировщик
├── parser.py         # Парсинг страницы Avito
├── storage.py        # Хранение данных (пользователи, отправленные объявления)
├── config.py         # Конфигурация (токен, интервал)
├── requirements.txt  # Зависимости Python
├── user_data.json    # Создаётся автоматически — ссылки и статусы пользователей
└── seen_ads.json     # Создаётся автоматически — уже отправленные объявления
```

---

## ⚙️ Установка и запуск

### 1. Требования

- Python 3.10+
- pip

### 2. Установка зависимостей

```bash
pip install aiogram httpx beautifulsoup4 apscheduler lxml
```

> Если возникают конфликты версий — не указывай версии явно, pip сам выберет совместимые.

### 3. Настройка `config.py`

Открой файл `config.py` и вставь свой токен:

```python
BOT_TOKEN = "123456789:AAFxxxxxxxxxxxxxxxxxxxxxxxx"
CHECK_INTERVAL = 300  # интервал проверки в секундах (300 = 5 минут)
```

**Как получить токен:**
1. Открой Telegram и найди [@BotFather](https://t.me/BotFather)
2. Отправь команду `/newbot`
3. Придумай имя и username боту
4. Скопируй выданный токен в `config.py`

### 4. Запуск

```bash
python main.py
```

## 🤖 Использование бота

### Шаг 1 — Запуск
Открой бота в Telegram и отправь `/start`. Появится меню с кнопками.

### Шаг 2 — Установить ссылку
1. Открой [avito.ru](https://www.avito.ru) в браузере
2. Настрой фильтры поиска (категория, город, ценовой диапазон и т.д.)
3. Скопируй URL из адресной строки
4. Нажми кнопку **🔗 Установить ссылку** в боте
5. Вставь скопированную ссылку

### Шаг 3 — Запустить мониторинг
Нажми **▶️ Запустить мониторинг** — бот сразу сделает первую проверку и начнёт присылать новые объявления каждые 5 минут.

### Кнопки меню

| Кнопка | Описание |
|--------|----------|
| 🔗 Установить ссылку | Задать URL страницы поиска на Avito |
| ▶️ Запустить мониторинг | Начать отслеживание новых объявлений |
| ⏹ Остановить | Остановить мониторинг |
| 📋 Статус | Показать текущую ссылку и статус мониторинга |

## 📂 Описание файлов

### `config.py`
Хранит конфигурацию бота.

| Параметр | Описание |
|----------|----------|
| `BOT_TOKEN` | Токен Telegram-бота, полученный от @BotFather |
| `CHECK_INTERVAL` | Интервал проверки в секундах. По умолчанию `300` (5 минут) |

---

### `main.py`
Основной файл. Содержит:

- **`SetupStates`** — FSM-состояния для диалога ввода ссылки
- **`main_keyboard()`** — главное меню бота с кнопками
- **`check_and_notify(chat_id, url)`** — парсит Avito, находит новые объявления, отправляет их пользователю и сохраняет в историю
- **Хэндлеры:**
  - `/start` — приветствие
  - `🔗 Установить ссылку` — переводит в состояние ожидания ввода URL
  - `SetupStates.waiting_for_url` — принимает URL, валидирует (проверяет наличие `avito.ru`), сохраняет
  - `▶️ Запустить мониторинг` — добавляет задачу в планировщик, запускает первую проверку
  - `⏹ Остановить` — удаляет задачу из планировщика
  - `📋 Статус` — выводит текущую ссылку и статус
- **`main()`** — запускает планировщик, восстанавливает активные мониторинги после перезапуска, запускает polling

---

### `parser.py`
Парсит страницу поиска Avito.

**Функция `parse_avito(url) -> List[Dict]`**

- Делает GET-запрос с заголовком User-Agent (имитация браузера)
- Находит карточки объявлений по селектору `div[data-marker='item']`
- Из каждой карточки извлекает:
  - `title` — название объявления (`h3[itemprop='name']`)
  - `price` — цена (`meta[itemprop='price']`)
  - `link` — ссылка (`a[data-marker='item-title']`)
- Возвращает список словарей `[{"title": ..., "price": ..., "link": ...}]`

> ⚠️ **Важно:** Avito периодически меняет HTML-разметку. Если парсер перестал работать — обнови CSS-селекторы в `parser.py`, открыв страницу через инструменты разработчика в браузере (F12).

---

### `storage.py`
Управляет хранением данных в JSON-файлах.

**Функции для отправленных объявлений:**

| Функция | Описание |
|---------|----------|
| `load_seen(chat_id)` | Загружает множество уже отправленных ссылок для пользователя |
| `save_seen(chat_id, seen)` | Сохраняет обновлённое множество ссылок |

**Функции для данных пользователей:**

| Функция | Описание |
|---------|----------|
| `load_user_data()` | Загружает словарь всех пользователей `{chat_id: {url, active}}` |
| `save_user_data(data)` | Сохраняет обновлённые данные пользователей |

**Структура `user_data.json`:**
```json
{
  "123456789": {
    "url": "https://www.avito.ru/moskva/...",
    "active": true
  }
}
```

**Структура `seen_ads.json`:**
```json
{
  "123456789": [
    "https://www.avito.ru/item/123",
    "https://www.avito.ru/item/456"
  ]
}
```

## 🔄 Логика работы

```
Пользователь нажимает ▶️ Запустить
          │
          ▼
APScheduler добавляет задачу check_and_notify()
(запускается каждые 5 минут)
          │
          ▼
     parse_avito(url)
   Парсит страницу Avito
          │
          ▼
  Фильтр по seen_ads.json
  Оставляем только новые
          │
     ┌────┴────┐
  Новых нет   Есть новые
     │              │
  Пропускаем   Отправляем в Telegram
                    │
                    ▼
           Сохраняем ссылки
           в seen_ads.json
```

## 🛠 Возможные проблемы

### Бот не реагирует на сообщения
- Убедись, что `BOT_TOKEN` в `config.py` правильный
- Проверь, что скрипт запущен: `python main.py`

### Парсер ничего не находит
- Проверь вручную — открой ссылку из `user_data.json` в браузере, убедись, что объявления есть
- Возможно, Avito обновил разметку — открой страницу через F12 и найди актуальные селекторы для карточек

### Ошибка `ResolutionImpossible` при установке
Не указывай версии пакетов. Вместо `requirements.txt` используй:
```bash
pip install aiogram httpx beautifulsoup4 apscheduler lxml
```

### Бот перестал слать уведомления после перезапуска
Это нормальное поведение при `active: false` в `user_data.json`. Просто нажми **▶️ Запустить мониторинг** снова — либо отредактируй `user_data.json` вручную, поставив `"active": true`.

## 🚀 Деплой на сервер (опционально)

Чтобы бот работал постоянно, запусти его на VPS через `systemd` или `screen`.

**Через screen (быстро):**
```bash
screen -S avitobot
python main.py
# Ctrl+A, затем D — отсоединиться, бот продолжит работу
```

**Через systemd (надёжно):**

Создай файл `/etc/systemd/system/avitobot.service`:
```ini
[Unit]
Description=Avito Telegram Bot
After=network.target

[Service]
WorkingDirectory=/путь/к/avito_bot
ExecStart=/usr/bin/python3 main.py
Restart=always

[Install]
WantedBy=multi-user.target
```

Затем:
```bash
systemctl enable avitobot
systemctl start avitobot
systemctl status avitobot
```
